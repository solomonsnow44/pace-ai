# PaceOps Frontend Build Brief

Build the first professional CRM frontend for PaceOps.

This is a multi-client account management CRM for PaceOps and later licensed customers.

The first user story:
Solomon and Amer are working on the Each&Other client account.
Last week they ran a survey outreach campaign.
This week they are targeting accounts like Stripe and Santander.
They need to switch between clients, campaigns, accounts, contacts, and call workflows quickly.

The UX should support:
- client switching
- campaign switching
- accounts list
- contacts list
- pipeline kanban
- call actions
- research panel
- AI helper drawer
- CSV import entry point
- integration placeholders

Build with mock data first.

Pages to create:
1. Dashboard
2. Clients
3. Client detail
4. Campaigns
5. Campaign detail
6. Accounts
7. Account detail
8. Contacts
9. Contact detail
10. Pipeline board
11. Calls
12. Research
13. Files
14. Integrations
15. Settings

Important:
- Use light/dark mode.
- Use the PaceOps logo from the root/images folder.
- Use lucide-react icons.
- Use professional spacing and typography.
- Use a HubSpot-inspired layout but not HubSpot branding.
- AI should be a subtle assistant panel, not the whole product.
- Make it feel production-grade.

Aircall MVP:
- Phone numbers should render as clickable call buttons.
- Use tel: links initially.
- Create placeholder UI for future Aircall connection.
